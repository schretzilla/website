Project Smart Egg

February/2018

Why do we need such a smart egg?

Traditionally, in an egg drop contest, the objective is to build a device that will safely carry an egg from a multi-story drop to the ground. As you can imagine, the results of this competition are very binary. The egg either breaks, or it doesn’t, but where’s the fun in that? In order to make things a little more interesting the Fablab held a “Hatchathon” one afternoon that introduced me to a great group of dedicated engineers.

At the Yolk of the Smart Egg:

Using an ESP32 with an accelerometer we were able to develop a “Smart Egg.” One of the great things about the ESP32 is that it can be used as a Wi-Fi hotspot. By hosting an asynchronous web server on the ESP32 I was able to create a controller for the accelerometer readings. Integrating with the ESP32’s API, a connected user could record multiple drops and graph their data. To graph the data I used the Highchart’s framework, which allowed for some pretty detailed analysis of the eggs accelerometer readings. However, in case the user wanted to do more research, I also created an ability to download the recorded data so that further analysis could be done post drop on the user’s own device.

Takeaways:

Overall this was one of my favorite projects to date. I’ve never worked with such a diverse group of talent before. There were definitely a lot of things I would have done differently had I known what I knew now, however that’s kind of the fun part about it. This was the first time I’ve seen a full project transition from an idea on paper all the way to a completed physical object. This small afternoon project turned into a 2-month side project and the end result was the smartest egg on the planet (self proclaimed but I’m thinking competition isn’t’ too tough here).

My Eggheads:

Together these guys and I managed to design, create, and deploy 25 smart eggs for students competing in E-Week:

1. Garret Gray was able to write up the C++ code to read from the accelerometer

2. Andrew Miyaguchi worked with the ESP32’s asynchronous webserver as well as API.

3. Oscar Castillo used his design and modeling mastery to design a beautiful 3D printed egg that simply screws together with no extra parts required.

4. Zach Levine did a good amount of solderng and put together much needed documentation for the egg.

Tools:

  • ESP32
  • Highcharts JS
  • A 3D printed egg
  • Bluma's CSS Framework
